Events

All the logic of your game will be inside the event handlers, which are functions that belong to an object and are executed when some event happens. They may modify the object itself and create more events, thus altering the state of your game.

These are the three basic events that are received automatically by every object and you can implement:

  • #setup: Every object receives this event when it is spawned
  • #loop: This event is executed on every object on every frame
  • #collide: Whenever a collision is detected, this event is received on the objects colliding
  • #exit: This event is executed on an object before it is destroyed (manually or by loading a new level)

When an object receives an event, if it has the corresponding event handler it will be executed. An event handler looks like this: 

        {
            name: "#setup", code: function (data) {
                //This code is executed when the object is spawned
            }
        }

As you can see, it is a JavaScript function inside which you can do whatever you want, and the event might have some data associated that you will receive as the argument.

In order to modify the object or interact with the engine you can use the Object API.
