Manifest

The manifest is a JSON file that can be found at the root of any project and contains basic information about the game. Here you have the manifest from the starter project: 
    {
    "name": "gameName",
    "scope": "gameFiles",
    "components": [
        "components.js"
    ],
    "levels": [
        "levels.xml"
    ],
    "spritesheets":[
        "spritesheets.xml"
    ],
    "screenResolution": {
        "w": 1366,
        "h": 768
    },
    "dependencies": {},
    "themeColor":"#4B5",
    "backgroundColor":"#333",
    "tileIcon":"tileIcon.png",
    "enginefps": 60,
    "animationfps": 60
}
These are the different properties of the manifest:
  • name: the name of the game.
  • scope: the subfolder where the game files are located. Every file inside will be included in the game package.
  • components: the list of the .js files containing the components.
  • levels: the list of the files containing the levels, they can be in either .xml or .json files.
  • spritesheets: the list of the files containing the spritesheets, they can be in either .xml or .json format.
  • screenResolution: the resolution at which the game will be rendered.
  • dependencies: the dependencies that your game needs, they will be pulled from the Clockwork Package Manager.
  • themeColor: the color that should be used in your game icon, listings...
  • backgroundColor: the background color udisplayed behind your game when it is running.
  • tileIcon: a png image used as the icon for your game.
  • enginefps: the number of times per second the #loop event will happen in the engine.
  • animationfps: the number of times per second the game will be rendered.
